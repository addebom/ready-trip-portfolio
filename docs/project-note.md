# Project Note - 준비해여행

## 프로젝트 개요

준비해여행은 여행 준비 과정에서 반복적으로 발생하는 준비물 누락 문제를 해결하기 위해 개발한 여행 준비 체크리스트 서비스입니다.

사용자가 여행 상황에 맞는 준비물을 확인하고 관리할 수 있도록 설계했으며, 실제 서비스로 출시하여 운영 경험까지 확보하는 것을 목표로 진행했습니다.

---

## 문제 정의

여행을 준비할 때 많은 사람들이 비슷한 문제를 반복적으로 경험합니다.

* 무엇을 챙겨야 하는지 매번 다시 생각해야 한다.
* 출발 직전 준비물을 빠뜨리는 경우가 발생한다.
* 출국 준비와 귀국 준비가 혼재되어 관리가 어렵다.
* 항공사나 여행 조건에 따라 필요한 준비물이 달라진다.

기존에는 메모장이나 메시지 앱을 이용해 준비물을 관리하는 경우가 많았지만, 반복 사용성과 관리 측면에서 한계가 있다고 판단했습니다.

---

## 목표

이 프로젝트의 목표는 단순한 체크리스트를 만드는 것이 아니었습니다.

사용자가 여행 상황에 따라 필요한 준비물을 쉽게 확인하고 관리할 수 있도록 하며, 최소한의 입력만으로 준비 과정을 정리할 수 있는 서비스를 만드는 것을 목표로 했습니다.

---

## 주요 기능

### 여행 준비 체크리스트

사용자가 준비물을 추가하고 완료 여부를 관리할 수 있도록 구성했습니다.

### 출국 / 귀국 분리 관리

가는 날과 돌아오는 날에 필요한 준비물을 별도로 관리할 수 있도록 설계했습니다.

귀국 탭에서는 출국 시 등록한 준비물 목록을 기반으로 다시 확인할 수 있도록 구성했습니다.

### 추천 준비물 제공

사용자가 직접 모든 준비물을 입력하지 않아도 여행 상황에 맞는 추천 항목을 확인할 수 있도록 구현했습니다.

### 중복 입력 방지

오타나 유사한 표현으로 인해 같은 준비물이 중복 등록되는 문제를 방지하기 위해 입력값 정규화 및 중복 확인 로직을 적용했습니다.

### 모바일 사용 환경 최적화

토스 미니앱 환경을 고려하여 터치 기반 사용 흐름과 모바일 화면 중심 UI를 설계했습니다.

---

## 기술적 의사결정

### 단일 화면 중심 구조

사용자가 복잡한 화면 이동 없이 필요한 기능에 접근할 수 있도록 단순한 구조를 유지했습니다.

### 추천 기능 우선 적용

많은 기능을 추가하기보다 실제 사용 빈도가 높을 것으로 예상되는 추천 기능과 체크리스트 기능에 집중했습니다.

### 정책 기반 기능 조정

토스 미니앱 정책을 검토하며 외부 이동 중심 기능보다 인앱 경험을 유지하는 방향으로 기능을 수정했습니다.

## 서비스 운영 결과

* 실제 서비스 출시
* 출시 후 약 1개월 운영
* 누적 사용자 약 110명
* 일 사용자 1명 ~ 7명 수준

작은 규모였지만 실제 사용자가 서비스를 이용하는 과정을 관찰할 수 있었고, 개발 이후 운영 단계의 중요성을 경험할 수 있었습니다.
